## Status

## Decisions

- **F-22**: mobile booking week-nav chevrons are RTL-aware via a
  `WeekShiftButton` copying clinic.tsx's `backward !== I18nManager.isRTL`
  pattern exactly.
- **F-23**: all 8 hardcoded placeholder literals (4 web + 4 mobile)
  replaced with catalog usage — the existing `web.auth.phoneHint` /
  `web.book.phoneHint` for the seven phone fields, one new key
  `web.book.dobPlaceholder` ×3 locales for the DOB field
  (deliberately "YYYY-MM-DD" in all three — a literal LTR-pinned ISO
  input format). Repo sweep: zero `placeholder="` literals remain.
- **F-24**: the mobile consumed-keys guardrail ported to
  `apps/web/test/i18n-consumed-keys.test.ts` with next-intl coverage:
  `useTranslations`, positional `await getTranslations("ns")`, and the
  object `getTranslations({ locale, namespace })` form; dynamic
  templates expand fail-closed against contracts enums (booking
  status/action, clinical source/prescription, ROLES, PROVIDER_TYPES),
  the legal pages' `${section}Title|Body` against the same file's
  SECTIONS literal, and the locale-switcher's whole-key `t(locale)`
  against LOCALES. **311 consumed keys extracted; liveness floor 150.**
  Red-proof: an injected unknown key failed naming key + file in all
  three locales; reverted clean.
